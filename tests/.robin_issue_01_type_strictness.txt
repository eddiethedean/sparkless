## Summary

With **robin-sparkless 0.6.0**, comparing a string column to an integer literal in a filter raises `RuntimeError: cannot compare string with numeric type (i32)`. PySpark coerces and returns matching rows.

Related: #201 (closed).

## PySpark (expected)

```python
from pyspark.sql import SparkSession, functions as F
spark = SparkSession.builder.master("local[1]").getOrCreate()
df = spark.createDataFrame([{"str_col": "123"}, {"str_col": "456"}])
out = df.filter(F.col("str_col") == F.lit(123)).collect()
# Returns 1 row (PySpark coerces "123" for comparison)
spark.stop()
```

## Robin 0.6.0 (actual)

```python
import robin_sparkless as rs
F = rs
spark = F.SparkSession.builder().app_name("repro").get_or_create()
data = [{"str_col": "123"}, {"str_col": "456"}]
schema = [("str_col", "string")]
df = spark._create_dataframe_from_rows(data, schema)
out = df.filter(F.col("str_col") == F.lit(123)).collect()
# RuntimeError: cannot compare string with numeric type (i32)
```

## Expected

Allow string vs numeric comparison with coercion (PySpark parity), or document and expose a config to enable coercion.

## Reproducer

Run from sparkless repo: `python scripts/repro_robin_limitations/01_type_strictness.py` (with robin-sparkless>=0.6.0 and pyspark installed).
